# User-centred design

All work on the NHS App should follow the [Service Standard](https://service-manual.nhs.uk/standards-and-technology/service-standard). 

The services included in the App should be built using the [NHS App Design System](https://design-system.nhsapp.service.nhs.uk/), which is a small extension of the [NHS Design System](https://service-manual.nhs.uk/design-system). 

Beyond that, there are a number of ways of working and tools that we use to make sure we are designing good services that meet user needs and are consistent with the rest of the app:

- [Communities of practice](ucd-communities-of-practice.md)
- [Design assurance reviews](../governance/design-assurance.md)
- [Information architecture](information-architecture.md) ([and making changes to it](making-IA-changes.md))
- [Design system](design-system.md)
- [Prototyping tools and resources](prototyping-tools.md)
- [User experience principles](user-experience-principles.md)
- [User experience metrics](user-experience-metrics.md)
- [User needs](user-needs.md)
- [Accessibility audits](accessibility-audits.md)
- [Content governance](content-governance.md)
- [UCD ops](ucd-ops.md)
