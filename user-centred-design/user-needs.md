## Background

User needs are stored within the User Research Repository within Jira. There are dashboards within Confluence which give an overview of the repository of user needs we have documented:

- User research repository (Jira): Includes findings and user needs
- User needs tracker (Confluence): This tracks the user needs held within the repository

For access to the two tools listed above, please speak to the App's [UCD Ops team](ucd-ops.md).

## Common user needs

We have reviewed all user needs (as of March 2026) and have summarised the user needs into a set of common user need themes. These themes help us understand what is core to the app for our users. These are:

| Theme | Description | User need |
| ----- | ----------- | --------- |
| **Access to data** | Providing access to data within the app either on their own health, that of others, or that of services they are using. Having this data at their fingertips can support them to understand their healthcare or complete tasks which they need to do | As a user, I need access to information about my healthcare or the services I am using, so that I can understand and use this information for managing my health. |
| **Accessibility** | Users needing additional support in order to use the app for those with access needs | As a user with access needs, I need the services I use to be accessible, so that I am able to use them myself without having to ask for help | 
| **Actions** | Users performing an action either in or outside the app | As a user, when I am required to complete tasks to progress my healthcare, I need to be able to complete these actions, so that I can receive the care I need |
| **Clarity of information** | Information presented to them to be clear and understandable, removing jargon and avoiding the need to seek further support to understand what they are being presented | As a user, I need the information I receive to be clear and understandable, so that I am confident in my understanding and if I need to do anything |
| **Continuation of care** | Providing information within the app to support the user's care journey and ensure its consistency across care settings and people they speak with | As a user, I need my information to be passed from one person to the next within my healthcare journey, so that it is consistent and I don't have to repeat myself to lots of people |
| **Finding information** | Being able to find information related to their health or others'. This can also be related to information about services, medication, or care settings. This is closely related to 'Access to data' but is more around the action of locating it | As a user, I need to be able to find information about my health, so that I can use it when required |
| **Navigation** | Supporting users in getting to the right service or part of the app | As a user, I need to be able to find information or services when I need clarity or to support a decision, so that I get to what I am looking for | 
| **Onboarding** | Learning about the app, how to use it, and the services within it | As a user, I need to be informed about the NHS App, what it can do, how to use it, and about the services within it, so that I know what I can use it for and how |
| **Predictability** | Providing consistency within the app in terms of service availability providing users with a predictable service overall in terms of knowing what they will experience in the app. Without this it creates an unpredictable experience of not knowing what you will encounter when you open the app and use the services | As a user, I need consistently available features within a service, so that I have a predictable experience of feature availability and know which channel to use for which task |
| **Preferences** | Ensuring that the app adheres to their choices | As a user, I need my preferences to be met for the services that I use, so that my needs are met to my liking |
| **Seeking Help** | Supporting users in the ability to seek support when they have questions, need to complete a task or want extra information | As a user, I need to be able to find help or support when I need it, so that my problem can be resolved |
| **Trust** | Providing a trustworthy service for users | As a user, I need the services I use to be trustworthy, so that I am confident in using them without risk |
| **Updates** | Being updated on a change, something new, or being reminded about something related to their healthcare or the app itself | As a user, I need to be updated about a change or something new related to my healthcare, so that I am well informed <br/><br/>As a user, I need to be reminded about important events in my care, so that I don't miss them |
